# AI Card Generation View ŌĆö UX Fix Plan

## Problem statement
* In `/generate` the Model select shows only "Default for provider" on first paint. Models appear after client effect, bad UX.
* Spinner during card generation is raw `<div class="animate-spin...">` with no accessible label, no shared component, duplicated across flows. User reports "no UI indicator besides a circle".
* Root cause: `useAvailableModels` uses `useAction` client fetch on mount, loading starts false then true, causes flash. Generation views should not client-fetch at all ŌĆö models are user-scoped and cheap to preload server-side.

## Guiding decisions
* Do not reuse `useAvailableModels` hook in generation flows. It couples to `useAction`, triggers extra fetch, loading mechanism is broken for first paint.
* Server preload via `fetchAvailableModelsForServer(token)` is source of truth, same pattern already used in study/browse/flagged pages for Assistant panel.
* AiModelField and AiRefinementPanel become pure presentational: receive `models: readonly LlmModel[]` as prop, no hook, no effect, no loading state.
* Wizard AiPath is step 2, not first paint ŌĆö still preload server side in `/sets/new` page, pass down, no deferred fetch. If we later want lazy, wrap AiPath in Suspense boundary, but initial implementation preloads to keep simple.
* Create shared `Spinner` UI component for accessible loading indicator.

## Phase 1 ŌĆö Shared Spinner component
Files: new `src/components/ui/Spinner.tsx`
* Props: `size?: 'sm'|'md'|'lg'`, `label?: string`, `className?`
* Render: `<span role="status" aria-live="polite" aria-label={label||'Loading'} className="inline-flex items-center gap-2"> <span className="animate-spin rounded-full border-2 border-accent border-t-transparent ..." /> {label && <span className="text-sm text-muted">{label}</span>} </span>`
* Size map: sm h-4 w-4 border-2, md h-6 w-6 border-2, lg h-8 w-8 border-4
* Replace ad-hoc divs in:
  - `src/app/generate/GenerateInner.tsx` loading step
  - `src/app/sets/[setId]/AiAppendFlow.tsx` generating/confirming phase
  - `src/components/wizard/AiPath.tsx` generating block
  - other 5 usages found via grep for consistency: ExploreClient, StudySessionInner, etc (out of scope but follow same component for consistency, optional)
* Commit: `ui: add Spinner component with a11y and replace ad-hoc spinners`
