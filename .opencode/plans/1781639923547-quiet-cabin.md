# AI Card Generation Spinner Fix Plan

## Problem
Within `/generate` AI Card Generation view, loading state shows static circle not spinning. Root cause: ad-hoc `<div className="animate-spin h-8 w-8 border-4 border-accent border-t-transparent rounded-full" />` duplicated across codebase, no accessible label, no shared component, border spinner has low contrast and relies on Tailwind animate-spin which is fragile in Tailwind v4 setup. User reports "no UI indicator besides a circle".

## Goals
* Replace ad-hoc spinner divs with shared accessible SVG Spinner component.
* Ensure visible animation in AI Card Generation loading step and all generation flows.
* Add role="status", aria-label, optional visible label text.
* Consistent sizes sm md lg across UI primitives.

## Non-Goals
* No model preload changes — prior plans covered that separately, out of scope for this spinner fix.
* No backfill needed.

## Approach

**Phase 1 — Create src/components/ui/Spinner.tsx**
* New file exporting `Spinner({ size?: 'sm'|'md'|'lg'|'xl', label?: string, className?: string, inline?: boolean })`
* Render SVG spinner with Tailwind `animate-spin` on SVG element for reliability: circular path with stroke-accent, stroke opacity for track, strokeDasharray for spinner arc. Ensures visible motion even if border utilities misbehave.
* Sizes: sm 16px border-2 equivalent, md 24px, lg 32px, xl 40px. Map to h-4 w-4, h-6 w-6, h-8 w-8, h-10 w-10.
* Accessibility: outer span `role="status" aria-live="polite" aria-label={label || 'Loading'}`. If label prop provided, render adjacent text span with `text-sm text-muted`. If inline false, center flex column.
* Export from src/components/ui index if exists, else direct import.

Atomic commit: `ui: add accessible SVG Spinner component`
