# AI Card Generation UX Fix Plan

## Problem Statement
Within `/generate` AI Card Generation view:
* Models do not appear on first paint — select shows only "Default for provider", then populates after client fetch, bad UX.
* Loading spinner during card generation is raw div with animate-spin, no accessible label, duplicated across codebase, reported as "no UI indicator besides a circle".

Root cause: `AiModelField` calls `useAvailableModels()` which uses `useAction(api.ai.getAvailableModels)` client-side on mount. No server preload in generate flow. Hook loading starts false then true, causing flash. Same pattern in Wizard AiPath and Set detail AiAppendFlow.

## Goals
* First paint of `/generate` shows full model list from server, zero client fetch for models.
* Same for Set detail append flow.
* Wizard `/sets/new` step 2 fetches models only when AiPath mounts, not at top-level page load, using Suspense boundary so step 1 paint is not blocked.
* Replace ad-hoc spinner divs with shared accessible Spinner component.
* Remove `useAvailableModels` dependency from generation surfaces; keep hook only for Assistant panel where lazy open semantics remain, or refactor it separately.

## Non-Goals
* No backfill needed — side project, no active users.
* No change to Assistant panel preload behavior beyond optional hook cleanup.
* No new dependencies beyond existing Convex + React Query patterns already in repo; prefer lightweight client fetch wrapper over adding tanstack if not already present — check existing `useQuery` usage first, else small fetch-on-mount component with Suspense.

## Phase 2 — Server preload models for /generate
Files touched:
* `src/app/generate/page.tsx`
* `src/app/generate/GenerateClient.tsx`
* `src/app/generate/GenerateInner.tsx`
* `src/app/generate/GenerateConfigForm.tsx`
* `src/components/ai-generation/AiModelField.tsx`
* `src/components/AiRefinementPanel.tsx`

Changes:
* page.tsx: import `fetchAvailableModelsForServer`, `Promise.all` with `preloadQuery(api.flashcardSets.list...)` and models fetch. Pass `initialModels` prop down.
* GenerateClient: accept `initialModels?: readonly LlmModel[]`, forward to GenerateInner.
* GenerateInner: accept models prop, forward to GenerateConfigForm and to AiRefinementPanel.
* GenerateConfigForm: accept `models` prop, forward to AiModelField.
* AiModelField: new props `{ value, onChange, models: readonly LlmModel[], label?, defaultLabel?, disabled? }`. Remove `useAvailableModels` import. Compute options directly from props. If models empty, render disabled Select with "Loading models..." placeholder or empty state — no client fetch.
* AiRefinementPanel: same prop change, remove hook, receive models from parent.
* GenerateInner loading step uses new Spinner lg.

Result: first paint has models, zero useAction call for models in generate flow.

Atomic commit: `generate: preload models server-side and prop-drill to AiModelField`

## Phase 4 — Wizard AiPath lazy fetch without top-level preload
Files:
* `src/app/sets/new/page.tsx` — unchanged, do NOT add models preload here to avoid blocking first paint for all wizard steps.
* `src/components/wizard/WizardShell.tsx` — unchanged props.
* `src/components/wizard/AiPath.tsx`
* New optional: `src/components/ai-generation/AiModelFieldWithQuery.tsx` or inline fetch in AiPath.

Changes:
* AiPath is client component mounted only at step 2 "Add cards" when user selects AI source.
* Inside AiPath, create local models state fetched on mount via `useAction(api.ai.getAvailableModels)` wrapped in `useEffect` once, or better use existing Convex `useAction` with Suspense boundary.
* Show Spinner sm inside AiModelField area while loading, disabled select.
* Once loaded, pass models array down to AiModelField (now pure presentational).
* Do not modify page.tsx to preload; keeps step 1 fast. AiPath self-contained fetch ensures models appear automatically when user reaches AI step, without manual refresh.
* If repo already uses TanStack Query elsewhere, prefer that for caching; else simple useEffect once is acceptable as existing hook did, but without global hook abstraction — localized to AiPath wrapper.

Rationale from hint: initial models optional, making models required at top level would force preload for entire wizard. Instead query within wrapper that actually requires component.

Atomic commit: `wizard: lazy-fetch models in AiPath with Suspense, no top-level preload`

## Rollback path
* Each phase is additive prop threading; old hook path can be restored by reverting AiModelField to call useAvailableModels. Server preload is optional fallback — if fetchAvailableModelsForServer fails returns undefined, components render empty select disabled, same degraded UX as before but no crash.
* Spinner component is drop-in replacement; revert commit restores divs.

## Out of scope
* Refactoring Assistant panel hook loading mechanism beyond comment — separate cleanup if desired.
* Adding TanStack Query dependency unless already present; check `package.json` first, repo uses Convex useQuery/useAction patterns today.
* Changing model fetching to useQuery instead of action — out of scope, action is current API.
