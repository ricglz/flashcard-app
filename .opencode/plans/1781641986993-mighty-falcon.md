# Available Models Context Plan

## Context
Current `useAvailableModels` hook fetches via `useAction` on mount causing flash and bad UX. `fetchAvailableModelsForServer` exists but only used in 3 pages for Assistant panel, not generation flows. `AiModelField` and `AiRefinementPanel` import hook directly, preventing SSR preload and forcing prop drilling workarounds in prior plans.

User direction confirmed:
- Add TanStack dependency acceptable
- Create context to avoid prop drilling
- SSR preload preferred, suspense fallback for conditional sub-components

Audit findings:
- Hook location: `src/hooks/useAvailableModels.ts`
- Server fetch: `src/lib/serverAiModels.ts`
- Consumers: `AiModelField`, `AiRefinementPanel`, `AssistantPanelInner`
- Missing preload pages: `/generate`, `/sets/new`, `/sets/[setId]`
- Existing preload pages: browse, session, flagged pass `initialAssistantModels` via props
- No `@tanstack/react-query` in package.json, only transitive core in lockfile

## Worktree strategy
Create new worktree to isolate changes:
```
git worktree add -b ai-models-context ../flashcard-app-wt-ai-models main
cd ../flashcard-app-wt-ai-models
```
All phases execute in this worktree. Rebase onto main before final PR, no merge commits per AGENTS.md. Route-level preload commits can be split to further worktrees `wt/ai-models-generate`, `wt/ai-models-wizard`, `wt/ai-models-setdetail` if parallelizing, then cherry-pick or rebase back to main worktree.

## Phase 1 — Dependency and root provider
- Add `@tanstack/react-query` to dependencies
- Create `src/lib/queryClient.ts` singleton QueryClient with suspense true, staleTime 5m, gcTime 10m
- Extend root layout or create `src/app/providers.tsx` client wrapper nesting `QueryClientProvider` inside existing Convex and Clerk providers, outside AI surfaces
- Verify no duplicate providers in test setup
- Atomic commit: `deps: add tanstack query and root provider`

## Phase 2 — AvailableModels context
- New `src/contexts/AvailableModelsContext.tsx` client:
  - `AvailableModelsProvider({ children, initialModels?: readonly LlmModel[] })`
  - Uses `useConvex()` client to call `client.action(api.ai.getAvailableModels,{})`
  - TanStack `useSuspenseQuery` with key `['ai','models']`, queryFn returns models array or empty on !ok, initialData from prop, staleTime Infinity
  - Export `useAvailableModelsContext(): readonly LlmModel[]`
- Keep `src/hooks/useAvailableModels.ts` as deprecated shim calling context for backward compat during migration, add comment pointing to context
- Atomic commit: `ai-models: add AvailableModelsContext with tanstack suspense`

## Phase 3 — Component refactor to context
- Update `src/components/ai-generation/AiModelField.tsx`: remove hook import, use context, no loading prop, empty array renders disabled select during suspense fallback
- Update `src/components/AiRefinementPanel.tsx`: same
- Update `src/components/AssistantPanelInner.tsx`: prefer context, keep initialModels prop optional for transition but ignore if provider present; remove direct hook call
- Update tests: `AiRefinementPanel.test.tsx` wrap with provider and supply mock context value instead of hook mock; add similar for AiModelField if needed
- Atomic commit: `ai-models: make AiModelField and AiRefinementPanel consume context`

## Phase 4 — Server preload per route
Add `fetchAvailableModelsForServer` and provider wrapper at page boundary, no prop drilling beyond provider prop:

- `src/app/generate/page.tsx`: Promise.all preloadQuery sets + fetch models, pass initialModels to GenerateClient which wraps its tree in AvailableModelsProvider
- `src/app/sets/new/page.tsx`: same for WizardShell
- `src/app/sets/[setId]/page.tsx`: same for SetDetailClient wrapping AiAppendFlow area
- Optional follow-up not blocking: migrate browse/session/flagged existing prop path to provider as well to fully remove initialAssistantModels prop drilling; note as tech debt if deferred
- Atomic commits per route:
  - `generate: preload available models server-side via context provider`
  - `wizard: preload available models server-side`
  - `set-detail: preload available models server-side`

## Phase 5 — Conditional suspense UI
- Assistant panel opens on click: ensure provider at page root means first open suspends once if no initialModels, then caches. Wrap model Select in Suspense fallback rendering disabled Select with Spinner sm to avoid flash.
- Verify no loading state flicker in generation flows because SSR provides initialData.
- Atomic commit: `assistant: use context suspense for lazy model load`
